# add-terms

코드블록에 등장하는 전문용어·명령어·옵션 중 설명이 빠진 것만 찾아 해설 bullet을 추가한다.

## 입력
- 대상 파일 경로 1개

## 실행 순서

1. **코드블록 수집** — 백틱 3개로 감싼 모든 블록을 추출.
2. **용어 후보 추출**:
   - 첫 단어가 명령어일 확률이 높음 (`git`, `docker`, `npm`, `cd`, `grep` 등)
   - `--xxx` / `-x` 플래그
   - `$VAR`, `${VAR}` 형태 환경변수
   - 백틱 안의 토큰
3. **문서 내 설명 존재 여부 검사**:
   - 용어가 본문(코드블록 외)에서 언급되고 있는가?
   - 표나 bullet에서 해설되고 있는가?
   - 둘 다 없으면 **"설명 누락"** 으로 분류.
4. **사용자에게 후보 제시**:
   - 누락 용어 리스트 + 각각의 1줄 제안 설명
   - 사용자가 confirm한 항목만 반영
5. **반영 위치**:
   - "라인별 해설" 섹션이 있으면 그 아래 bullet 추가
   - 없으면 해당 코드블록 바로 아래 bullet 추가
   - 양이 많으면 `## N. 용어 정리` 섹션 신설

## 금지
- 설명이 이미 있는 용어는 건드리지 않는다.
- 추정 설명 금지. 확실하지 않으면 `(설명 필요)` 플레이스홀더.
- 코드블록 내부 수정 금지. 오직 주변 해설만 추가.
