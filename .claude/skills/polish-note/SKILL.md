# polish-note

기존 노트를 원하는 6단 흐름에 맞춰 재구성한다.

## 레퍼런스
- 이상적인 구조 예시: `content/개발/git/5. git-hooks.md`
- 템플릿 기준: `content/templates/기본 문서 템플릿.md`

## 입력
- 대상 파일 경로 1개 (또는 현재 에디터 선택)

## 실행 순서

1. **현재 구조 분석**:
   - 섹션 제목 목록 추출
   - frontmatter 누락 필드 확인 (title, tags, date)
   - 서술형 종결(`~습니다`, `~합니다`) 카운트
2. **진단 보고** (2-5줄):
   - 현재 흐름이 6단에서 어느 단계가 빠졌는지
   - frontmatter / 문체 이슈
   - 재구성 플랜 요약
3. **사용자 승인** 후 재작성:
   - 기존 내용을 6단 섹션에 재배치 (내용 삭제 금지, 재배치만)
   - 부족한 섹션은 `(작성 필요)` 플레이스홀더로 표시
   - 간결체로 통일 — `~습니다` → `~이다`, `~합니다` → `~한다`
   - 코드블록은 원본 유지
4. **wikilink 검증** — `[[...]]` 링크가 실제 파일과 일치하는지 Glob으로 확인. 깨진 링크는 경고만.

## 금지
- 내용(지식) 자체를 임의 추가·삭제하지 않는다. **재구성과 문체만** 수정한다.
- 원본 코드블록 내용을 변경하지 않는다.
- 기술적 사실관계를 "추정"으로 채우지 않는다. 모르면 `(작성 필요)`.

## 출력
- 변경 diff 요약 (어느 섹션이 이동/신설/수정되었는지)
- 남은 작성 필요 항목 리스트
