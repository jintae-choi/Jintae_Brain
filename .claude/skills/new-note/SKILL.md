# new-note

content/ 하위에 학습 노트를 새로 만든다.

## 입력
- 카테고리 경로 (예: `개발/git`)
- 문서 제목 (예: `git rebase 정리`)
- 선택: 주제태그 리스트

## 실행 순서

1. **템플릿 로드** — `content/templates/기본 문서 템플릿.md` 읽기 (이미 원하는 6단 흐름 포함).
2. **frontmatter 치환**:
   - `title`: 문서 제목
   - `tags`: 카테고리태그 + 주제태그. 카테고리 폴더명(예: `git`)을 첫 태그로.
   - `date`: 오늘 날짜 (`YYYY-MM-DD`, 대화 컨텍스트의 현재 날짜 사용)
3. **파일 경로 결정**:
   - `content/{카테고리경로}/{번호}. {제목}.md`
   - 번호는 해당 폴더의 기존 `N. xxx.md` 패턴 중 가장 큰 번호 + 1
   - 번호 체계가 없는 폴더면 번호 생략
4. **파일 생성** — Write 도구로 템플릿 내용에 frontmatter만 치환해서 저장.
5. **index.md 갱신** — 같은 폴더의 `index.md`에 새 문서 링크(`[[N. 제목]] — 한 줄 요약`) 추가. index.md 없으면 스킵.
6. **확인 보고** — 생성한 경로와 다음 작업 제안(내용 작성 시작할지) 1-2줄로 보고.

## 금지
- 내용을 임의로 채우지 않는다. **템플릿 골격만** 생성한다. 사용자가 내용을 주면 그때 채운다.
- `quartz/`, `public/` 건드리지 않는다.
- frontmatter에 없는 필드 추가 금지.

## 문체
- 간결체 (`~이다.`, 명사형 종결). 서술형 지양.
