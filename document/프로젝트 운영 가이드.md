# Jintae Brain 프로젝트 운영 가이드

## 1) 프로젝트 목적
- Markdown 기반 개발 공부 문서를 누적 관리한다.
- 위키링크(`[[문서명]]`) 기반으로 지식 연결(백링크)을 강화한다.
- GitHub Pages에서 무료로 열람 가능한 정적 사이트를 운영한다.

## 2) 현재 환경/설정
- 경로: `d:\git\Jintae_Brain`
- 프레임워크: Quartz v4
- 런타임: Node.js 22 (GitHub Actions 기준)
- 호스팅: GitHub Pages + GitHub Actions
- 로케일: `ko-KR`
- baseUrl: `jintae-choi.github.io/Jintae_Brain`
- 폰트: Noto Sans KR (본문/헤더), IBM Plex Mono (코드)
- 주요 기능: SPA, Popover, Backlinks, Graph View, KaTeX, Syntax Highlighting

## 3) 설치/실행
1. 의존성 설치
   - `npm ci`
2. 로컬 빌드
   - `npx quartz build`
3. 로컬 미리보기
   - `npx quartz build --serve`

## 4) 배포 방식
- 워크플로우: `.github/workflows/deploy.yml`
- 트리거: `main` 브랜치 push 또는 수동 실행(workflow_dispatch)
- 배포 순서: build -> setup pages -> upload artifact -> deploy
- 사이트 URL: `https://jintae-choi.github.io/Jintae_Brain`

## 5) 콘텐츠 운영 규칙
- 문서 위치: `content/` 하위
- 카테고리: 폴더 단위 분리 (`개발/JavaScript`, `개발/DB` 등)
- Frontmatter 필수: `title`, `tags`, `date`
- 템플릿: `content/templates/기본 문서 템플릿.md`, `content/templates/TIL 템플릿.md`
- 태그 체계: 카테고리 + 주제 + 형식(TIL)

## 6) 토큰 절약 운영 규칙
- 기본 컨텍스트 제외: `.copilotignore` 유지
- 큰 파일/빌드 출력물/public 폴더는 분석 대상에서 제외
- 요청 범위 파일만 읽고, 필요한 줄 범위만 조회
- 변경 보고는 전체 재설명 대신 변경점 중심으로 작성
- 장시간 대화에서 주제가 바뀌면 새 세션 사용

## 7) 수정 우선 파일
- 설정: `quartz.config.ts`, `quartz.layout.ts`
- 배포: `.github/workflows/deploy.yml`
- 콘텐츠: `content/`
- 에이전트 지침: `.github/copilot-instructions.md`

## 8) 가급적 건드리지 않는 영역
- Quartz 코어: `quartz/` 내부
- 빌드 결과물: `public/`
