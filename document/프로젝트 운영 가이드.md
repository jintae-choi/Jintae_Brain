# Jintae Brain 프로젝트 운영 가이드

## 1) 프로젝트 목적
- Markdown 기반 개발 공부 문서를 누적 관리.
- 위키링크(`[[문서명]]`) 기반 지식 연결(백링크) 강화.
- GitHub Pages 정적 사이트 운영.

## 2) 환경/설정
- 프레임워크: Quartz v4
- 런타임: Node.js 22
- 호스팅: GitHub Pages + GitHub Actions
- 로케일: `ko-KR`
- baseUrl: `jintae-choi.github.io/Jintae_Brain`
- 폰트: Noto Sans KR (본문/헤더), IBM Plex Mono (코드)
- 주요 기능: SPA, Popover, Backlinks, Graph View, KaTeX, Syntax Highlighting

## 3) 실행 방법

### Docker (권장)
```bash
docker-compose up -d --build
# 접속: http://localhost:18080
# 종료: docker-compose down
```
> `content/` 수정 시 volumes 연결로 즉시 반영.

### Node 직접 실행
```bash
npm ci
npx quartz build --serve
# 접속: http://localhost:8080
```

## 4) 배포
- 워크플로우: `.github/workflows/deploy.yml`
- 트리거: `main` 브랜치 push 또는 수동 실행
- 사이트: `https://jintae-choi.github.io/Jintae_Brain`

## 5) 콘텐츠 운영 규칙

### 문서 위치/구조
- 문서 위치: `content/` 하위
- 카테고리: 폴더 단위 분리 (`개발/JavaScript`, `개발/DB` 등)
- 각 카테고리 폴더에 `index.md` 작성 (Quartz 폴더 페이지용)

### Frontmatter (필수)
```yaml
title: 문서 제목
tags:
  - 카테고리태그
  - 주제태그
date: YYYY-MM-DD
```

### 태그 체계
- 카테고리: JavaScript, Docker, git 등
- 주제: 클로저, 스코프, bare저장소 등
- 형식: TIL (해당 시)

### 문체 규칙
- 간결체 사용. `~이다.`, `~한다.`, 명사형 종결.
  - 예) "정상 작동.", "설정 필요.", "차단될 수 있음."
- 서술형(`~하는 것입니다`, `~할 수 있습니다`) 지양.
- 코드 블록에는 언어 태그 명시 (bash, js, ts 등).

### 템플릿
- `content/templates/기본 문서 템플릿.md` 1개로 통합 운영.
- TIL 작성 시: 불필요한 섹션 삭제, tags에 TIL 추가.
- 템플릿은 `ignorePatterns`에 포함되어 사이트에 노출되지 않음.

## 6) 수정 우선순위
1. 콘텐츠: `content/`
2. 설정: `quartz.config.ts`, `quartz.layout.ts`
3. 배포: `.github/workflows/deploy.yml`
4. AI 지침: `.github/copilot-instructions.md`

## 7) 건드리지 않는 영역
- Quartz 코어: `quartz/` 내부
- 빌드 결과물: `public/`

## 8) 토큰 절약 규칙
- 요청 범위 파일만 읽기. 전체 스캔 금지.
- 필요한 줄 범위만 조회.
- 변경점(delta) 중심 보고. 전체 재설명 금지.
- 장시간 대화에서 주제 전환 시 새 세션 권장.
