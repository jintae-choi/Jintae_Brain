# Jintae Brain 프로젝트 운영 가이드

> 실행·배포 방법은 [README.md](../README.md)를 참고한다. 이 문서는 **내부 운영 상세**에 집중한다.

## 1. 프로젝트 목적
- Markdown 기반 개발 공부 문서를 누적 관리.
- 위키링크(`[[문서명]]`) 기반 지식 연결(백링크) 강화.
- GitHub Pages 정적 사이트 운영.

## 2. 환경 / 스택 상세
- 프레임워크: Quartz v4
- 런타임: Node.js 22 (Docker 이미지 `node:22-slim`)
- 로컬 실행: **Docker 전용**. 기본/dev 2모드, 포트 `8080`. 상세: [[1. Docker 개발 모드]]
- 호스팅: GitHub Pages + GitHub Actions
- 로케일: `ko-KR`
- baseUrl: `jintae-choi.github.io/Jintae_Brain`
- 폰트 (Google Fonts 로드): IBM Plex Sans KR, Noto Serif KR, JetBrains Mono
- 디자인 테마 스위처 (A/B/C, 좌측 사이드바):
  - **A (기본) Technical Docs**: IBM Plex Sans KR (본문·헤더) + JetBrains Mono (코드), 딥블루 액센트, 플랫 레이아웃
  - **B Academic**: Noto Serif KR (본문) + IBM Plex Sans KR (헤더) + JetBrains Mono (코드), 시에나 톤, 카드형
  - **C Developer**: IBM Plex Sans KR (본문) + JetBrains Mono (헤더·코드), VS Code 다크톤, 마크다운 마커 헤더
  - 구현: `quartz/components/ThemeSwitcher.tsx`, `quartz/styles/custom.scss`. 상세: [[1. Quartz 디자인 테마 스위처]]
- 주요 기능: SPA, Popover, Backlinks, Graph View, KaTeX, Syntax Highlighting

## 3. 콘텐츠 운영 규칙

> 문체·frontmatter·구성 원칙 등 작성 규칙은 [CLAUDE.md](../CLAUDE.md)에 통합 관리.

### 문서 위치 / 구조
- 문서 위치: `content/` 하위
- 카테고리: 폴더 단위 분리 (`개발/JavaScript`, `개발/DB` 등)
- 각 카테고리 폴더에 `index.md` 작성 (Quartz 폴더 페이지용)

### 태그 체계
- 카테고리: JavaScript, Docker, git 등
- 주제: 클로저, 스코프, bare저장소 등
- 형식: TIL (해당 시)

### 템플릿
- [content/templates/기본 문서 템플릿.md](../content/templates/기본%20문서%20템플릿.md) 1개로 통합 운영.
- 6단 흐름(개요 → 전체 개념 → 사전지식 → 전체 코드 → 라인별 해설 → 경험/교훈)을 골격으로 고정.
- TIL 작성 시: 불필요한 섹션 삭제, tags에 TIL 추가.
- 템플릿은 `ignorePatterns`에 포함되어 사이트에 노출되지 않음.

### 레퍼런스 구현
- 이상적 구조의 살아있는 예시: [content/개발/git/5. git-hooks.md](../content/개발/git/5.%20git-hooks.md)
- 새 노트 작성 시 이 문서를 형식 기준으로 참고한다.
