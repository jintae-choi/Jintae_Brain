# 대화 회고 및 학습 기록

이번 대화에서 작업한 내용을 회고하고 학습 사항을 기록한다.

## 입력 (선택)
$ARGUMENTS

## 작업 절차

### 1단계: 대화 분석
이번 대화에서 다룬 내용을 분석한다:
- 어떤 문제를 해결했는가
- 사용자가 막혔거나 몰랐던 부분은 무엇이었는가
- 새로 배운 기술/개념/패턴은 무엇인가
- 시행착오가 있었다면 원인과 해결 방법

### 2단계: 학습 기록 작성
`docs/learnings/` 폴더에 날짜 기반으로 기록한다.
파일명: `YYYY-MM-DD-[주제].md`
- 제목은 핵심 행동과 결과가 드러나도록 구체적으로 작성한다 (예: "WBS 정규화와 뷰를 활용한 역정규화 전략", "Docker 익명볼륨→named볼륨 전환으로 패키지 동기화 해결")

```markdown
# [구체적 제목 — 무엇을 어떻게 했는지 드러나게]

- **날짜:** YYYY-MM-DD
- **프로젝트:** [프로젝트명]
- **분류:** [기술/도구/패턴/트러블슈팅]

## 문제/상황
- 무엇을 하려고 했는가
- 왜 이 작업이 필요했는가 (팀 요구, 기술적 한계, 기존 구조의 문제 등 의사결정 배경)

## 배운 것
(핵심 내용만 간결하게. 필요 시 코드/데이터 예시를 포함하여 나중에 봐도 바로 이해할 수 있도록)

## 기억할 포인트
(다음에 같은 상황에서 바로 적용할 수 있는 한 줄 요약)
```

**작성 원칙:**
- 나중에 이 문서만 보고도 당시 상황을 재현할 수 있어야 한다
- 구조 변경이 있었다면 Before/After 예시 데이터를 반드시 포함한다
- 의사결정의 이유(WHY)를 기록한다 — "무엇을 했다"보다 "왜 이렇게 했다"가 중요하다

### 3단계: 프로필 업데이트 확인
`docs/prompts/developer-profile.md`의 보유 기술 섹션에 변동이 필요한지 확인한다:
- 새로 사용하기 시작한 기술 → **사용 중**에 추가
- 충분히 익힌 기술 → **사용 중** → **학습 완료**로 이동
- 학습 이력 테이블 업데이트

변동이 없으면 건너뛴다.

### 4단계: 요약 보고
기록한 내용을 간략히 보고한다.
