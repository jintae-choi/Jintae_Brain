# 작업 방식 가이드

## 작업 원칙
- Agentic: 자율 계획-실행-검증, 체크포인트에서만 보고. 매 단계 확인 X
- 한국어로 응답. UI 텍스트도 한국어
- 새 기능/화면에 사용법 안내 포함
- 데이터 모델 설명 시 관계도 + 실제 예시 포함

## UI/UX 원칙
- 기술 용어 대신 사용자가 이해할 수 있는 맥락 있는 표현 사용
- 새 화면에는 사용법을 안내하는 UI 요소 포함

## 코드 품질 기준
- 구현 후 자기 검증: 빌드 통과 + 실제 동작 확인까지가 "완료"
- 테스트: 프로젝트 초기에는 수동 테스트 우선. 안정기 진입 후 핵심 로직 단위 테스트 추가

## 문서화 기준
- 기술 결정과 그 이유(why) → docs/에 기록
- README는 프로젝트당 1개, 실행 방법 중심

## Docker 환경 규칙
- Docker 볼륨: named 볼륨 사용 (익명 볼륨 X), npm install 선행 실행으로 패키지 동기화

## Claude Code 활용
- Claude Code 기능(메모리, TodoWrite, Agent, Plan 모드, 슬래시 커맨드, Hooks 등)을 적재적소에 사용
- 처음 사용하는 기능은 왜 쓰는지 한 줄 설명
