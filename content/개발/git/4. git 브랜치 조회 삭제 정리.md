# git 브랜치 조회 삭제 정리

## 개요
> 로컬/원격 브랜치 조회, 삭제, 동기화, 오류 원인 정리.

## 조회 명령어

### 로컬 브랜치 목록
```bash
git branch
```
- 로컬 저장소에 있는 브랜치 목록 확인.

### 원격 브랜치 목록
```bash
git branch -r
```
- 원격 저장소에 실제 존재하는 브랜치 목록 확인.

### 로컬 + 원격 전체 목록
```bash
git branch -a
```
- 로컬 브랜치와 원격 추적 브랜치 전체 확인.

### 최신 커밋 포함 조회
```bash
git branch -v
```
- 브랜치별 최신 커밋 해시와 메시지 확인.

### Tracking 상태 포함 조회
```bash
git branch -vv
```
- 로컬 브랜치가 추적하는 원격 브랜치와 ahead/behind 상태 확인.

### 패턴 필터 조회
```bash
git branch --list 'pattern*'
```
- 와일드카드 기준 브랜치 필터링.

### 병합 상태 기준 조회
```bash
git branch --merged
git branch --no-merged
```
- `--merged`: 현재 브랜치 기준 병합 완료 브랜치.
- `--no-merged`: 아직 병합되지 않은 브랜치.

## 삭제 명령어

### 로컬 브랜치 삭제(안전)
```bash
git branch -d 브랜치명
```
- 현재 브랜치에 병합된 브랜치만 삭제.

### 로컬 브랜치 강제 삭제
```bash
git branch -D 브랜치명
```
- 병합 여부 무시 후 강제 삭제.

### 원격 브랜치 삭제
```bash
git push origin --delete 브랜치명
```
- 원격 저장소의 브랜치 삭제.

### 원격 삭제 정보 로컬 정리
```bash
git fetch -p
```
- 원격에서 삭제된 브랜치의 추적 정보(prune) 정리.

## 왜 목록에는 보이는데 원격 삭제는 실패하는가

### 핵심 원인
- `git branch` 출력은 로컬 브랜치 기준.
- `git push origin --delete 브랜치명`은 원격 브랜치 대상.
- 로컬에만 있고 원격에 없으면 `remote ref does not exist` 발생.

### 점검 순서
```bash
git branch
git branch -r
```
- `git branch`에만 있고 `git branch -r`에 없으면 원격 삭제 대상 아님.

### 올바른 처리
```bash
# 로컬 브랜치 정리
git branch -d 브랜치명
# 필요 시 강제 삭제
git branch -D 브랜치명
# 원격 추적 정보 정리
git fetch -p
```

## 주의사항
- 현재 체크아웃된 브랜치는 삭제 불가.
- 삭제 전 `main` 또는 `dev` 등 다른 브랜치로 이동 필요.
- 강제 삭제 전 커밋 보존 필요 시 `git reflog` 확인 권장.
