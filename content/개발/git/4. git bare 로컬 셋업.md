# git bare 로컬 셋업

## 개요
> [[3. git bare 초기 준비]]에서 서버 인증·Bare 저장소 생성이 끝났음을 전제로 한다. 본 문서는 로컬 코드를 서버로 푸시하고, 서버 웹 디렉토리에 배포하고, 동료가 협업에 합류하는 과정을 정리한다. 모든 명령어는 Git Bash 기준이다.

## 환경 정보

| 항목 | 값 |
|---|---|
| 실서버 IP | `10.10.10.10` |
| 서버 계정 | `gitid` |
| 중앙 Bare 저장소 | `~/repos/jintae.git` |
| 서버 웹 디렉토리 | `~/www/jintae` |
| 로컬 개발 폴더 | `~/workspace/jintae` |

- **Bare 저장소**는 코드 보관·공유 전용 중앙 허브이며, 직접 편집 대상이 아니다. 개념과 생성 방법은 [[3. git bare 초기 준비]] 참조.
- **웹 디렉토리**는 Bare 저장소에서 최신 코드를 꺼내와 실제로 서버 프로세스가 구동되는 워킹 디렉토리이다.

---

## [STEP 1] 로컬 PC: 원격 등록 및 최초 푸시

로컬 코드를 서버 Bare 저장소로 전송한다.

```bash
# 1. 로컬 프로젝트 폴더로 이동
cd ~/workspace/jintae

# 2. 기존 origin 연결 제거 (없으면 조용히 무시)
git remote remove origin 2>/dev/null

# 3. 서버 Bare 저장소를 origin으로 등록
git remote add origin gitid@10.10.10.10:repos/jintae.git

# 4. main 브랜치 푸시 + upstream 연결
git push -u origin main
```

- `git remote remove origin 2>/dev/null` — 기존 origin이 없어도 에러를 무시한다. `2>/dev/null`은 에러 스트림만 버리는 리다이렉션이다.
- `-u` 옵션 — 로컬 `main`과 원격 `origin/main`을 tracking 관계로 연결. 이후부터 `git push` / `git pull`만 쳐도 된다.

---

## [STEP 2] 서버: 웹 디렉토리 배포 설정

Bare 저장소의 최신 코드를 실제 서비스가 구동되는 웹 디렉토리로 꺼내온다.

> **💡 왜 `git clone`을 안 쓰는가?**
> `git clone`은 **대상 폴더가 완전히 비어있을 때만** 동작한다. 웹 디렉토리에는 보통 `.env`, `ecosystem.config.cjs`, nginx 로그 등 Git 추적 대상이 아닌 설정 파일이 이미 들어있다. 이 상태에서 clone을 치면 "빈 폴더가 아니다"라며 실패한다. 따라서 아래 4단 콤보로 **기존 파일이 있어도 강제 덮어쓰기**를 수행한다.

```bash
# 1. 웹 디렉토리로 이동
cd ~/www/jintae

# 2. 소유권 보안 에러 회피 (Windows 다중 계정 환경)
git config --global --add safe.directory "$HOME/repos/jintae.git"

# 3. 강제 동기화 4단 콤보
git init                                    # 폴더를 Git 추적 대상으로 전환
git remote add origin ~/repos/jintae.git
git fetch origin                            # 최신 객체만 가져오기 (워킹 트리 미변경)
git reset --hard origin/main                # 워킹 트리를 origin/main 상태로 강제 덮어쓰기
```

- `git fetch` — 원격의 커밋·객체만 가져오고 워킹 트리는 건드리지 않는다.
- `git reset --hard origin/main` — 워킹 트리 내용을 무시하고 `origin/main` 상태로 덮어쓴다. 이 단계에서 기존 파일이 교체된다.

---

## [STEP 3] 동료: 로컬 환경으로 클론

새 개발자가 합류하는 경우, 동료의 로컬은 빈 폴더이므로 `git clone`을 그대로 쓸 수 있다.

```bash
# 동료 PC 터미널에서 실행
git clone gitid@10.10.10.10:repos/jintae.git
```

- Git은 원격 저장소 이름(`jintae.git`)에서 `.git`을 떼고 **현재 위치에 `jintae` 폴더를 자동 생성**한 뒤 그 안에 코드를 내려받는다. 현재 터미널 위치에 파일이 흩뿌려지지 않는다.
- 클론 직후 `origin`은 자동 등록되므로 별도 설정 없이 바로 `push` / `pull`이 가능하다.
