# git 브랜치 조회 삭제 정리

## 개요
> 로컬/원격 브랜치 조회, 삭제, 동기화, 오류 원인 정리.

## 자주 쓰는 명령어

```bash
git branch                       # 로컬 브랜치 목록
git branch -a                    # 로컬 + 원격 전체 목록
git branch -d 브랜치명            # 로컬 브랜치 삭제 (병합된 것만)
git branch -D 브랜치명            # 로컬 브랜치 강제 삭제
git push origin --delete 브랜치명 # 원격 브랜치 삭제
git fetch -p                     # 원격에서 삭제된 브랜치의 추적 정보 정리
```

## git branch 옵션 정리

| 옵션 | 설명 |
| --- | --- |
| (없음) | 로컬 브랜치 목록 |
| `-r` | 원격 브랜치 목록만 |
| `-a` | 로컬 + 원격 전체 |
| `-v` | 최신 커밋 해시/메시지 함께 표시 |
| `-vv` | 추적 원격과 ahead/behind 상태까지 표시 |
| `--merged` | 현재 브랜치에 병합된 것만 |
| `--no-merged` | 아직 병합되지 않은 것만 |
| `-d 브랜치명` | 병합된 브랜치 안전 삭제 |
| `-D 브랜치명` | 병합 여부 무시하고 강제 삭제 |

- 실무에서 체감상 가장 자주 쓰는 건 `git branch`, `git branch -a`, `git branch -d/-D` 정도.
- `-vv`와 `--merged`는 브랜치 정리 작업 시 주로 사용.
- `--list 'pattern*'`은 거의 쓰이지 않아 목록에서 제외.

## 왜 목록에는 보이는데 원격 삭제는 실패하는가

### 핵심 원인
- `git branch` 출력은 로컬 브랜치 기준.
- `git push origin --delete 브랜치명`은 원격 브랜치 대상.
- 로컬에만 있고 원격에 없으면 `remote ref does not exist` 발생.

### 점검 및 처리

```bash
git branch                  # 로컬에만 있는지 확인
git branch -r               # 원격에 존재하는지 확인 (없으면 원격 삭제 대상 아님)
git branch -d 브랜치명       # 로컬 브랜치 정리
git branch -D 브랜치명       # 필요 시 강제 삭제
git fetch -p                # 원격 추적 정보 정리
```

## 주의사항
- 현재 체크아웃된 브랜치는 삭제 불가.
- 삭제 전 `main` 또는 `dev` 등 다른 브랜치로 이동 필요.
- 강제 삭제 전 커밋 보존 필요 시 `git reflog` 확인 권장.
