git checkout dev
→ dev 브랜치로 이동합니다. 작업의 기준이 되는 브랜치를 명확히 지정합니다.

git pull
→ dev 브랜치의 최신 변경사항을 원격 저장소(origin)에서 받아와 로컬 dev 브랜치를 최신 상태로 만듭니다.

git checkout -b orbit_s
→ 현재 dev 브랜치의 최신 상태를 복사해 orbit_s라는 새 브랜치를 생성하고, 그 브랜치로 전환합니다.

git push -u origin orbit_s
→ 새로 만든 orbit_s 브랜치를 원격 저장소(origin)에 업로드하고, 로컬 orbit_s 브랜치와 원격 브랜치를 연결(tracking)합니다.

------------------------------

나중에 orbit_s를 dev 에 머지하고 싶으면 어떻게 해?

orbit_s 브랜치의 변경사항을 dev 브랜치에 머지하려면 다음과 같이 진행하면 됩니다.

dev 브랜치로 이동
git checkout dev

dev 브랜치를 최신 상태로 업데이트
git pull

orbit_s 브랜치를 dev에 머지
git merge orbit_s

충돌이 발생하면 충돌을 해결한 후
git add .
git commit

변경사항을 원격 dev 브랜치에 반영
git push


--------------------------------------


dev 브랜치의 기존 내용과 상관없이 orbit_s 브랜치의 코드를 dev 브랜치에 강제로 덮어쓰려면 다음과 같이 진행하면 됩니다.

dev 브랜치로 이동
git checkout dev

orbit_s 브랜치의 내용을 dev에 강제로 덮어쓰기
git reset --hard orbit_s

원격 dev 브랜치도 동일하게 덮어쓰기(주의: 기존 dev 이력 완전히 대체)
git push -f origin dev

이렇게 하면 dev 브랜치가 orbit_s 브랜치의 내용으로 완전히 대체됩니다.
(주의: 기존 dev 브랜치의 이력과 내용이 모두 사라지니 꼭 필요할 때만 사용하세요.)
