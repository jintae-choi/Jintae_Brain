# 개요

Quartz v4 기반 사이트의 좌측 사이드바에 **A/B/C 3가지 디자인 테마**를 즉시 전환할 수 있는 토글을 붙였다. 기본은 A(Technical Docs), 선택은 localStorage 저장으로 유지. 다크모드 토글과 독립 동작한다.

목적은 "디자인을 미리 단정하지 말고, 실제 콘텐츠 위에 3안을 올려놓고 비교해서 고르자"이다. Figma 시안만 보고 결정하기보다 실제 사이트에 올려 훨씬 정확한 판단이 가능하다.

# 전체 개념

3가지 디자인 톤:
- **A — Technical Docs**: 화이트 + 딥블루(#2563eb), IBM Plex Sans KR, 플랫(카드·그림자 제거), h1 하단 라인으로 위계 강화 — Stripe·Vercel·Linear docs 톤
- **B — Academic**: 따뜻한 오프화이트 + 시에나(#a0522d), Noto Serif KR 본문, 카드형 + 부드러운 그림자, 단락 들여쓰기, 이탤릭 blockquote
- **C — Developer Dark**: VS Code 톤(#1e1e1e + 민트 #4ec9b0), JetBrains Mono 헤더, h1/h2/h3 앞에 `#`·`##`·`###` 마크다운 마커, 샤프 코너

전환 방식은 **CSS 변수 오버라이드**다. Quartz가 기본으로 제공하는 `--bodyFont`·`--light`·`--secondary` 등을 `[data-design-theme="X"]` 선택자 안에서 다시 정의하면, 해당 attribute가 `<html>`에 붙는 순간 전체 사이트 테마가 바뀐다. 컴포넌트 코드는 하나도 안 건드린다.

전체 플로우:
1. ThemeSwitcher 컴포넌트가 `<button data-design="A|B|C">` 3개 렌더링
2. 페이지 로드 시 inline 스크립트가 localStorage에서 저장값 읽어 `<html data-design-theme="X">` 세팅 (기본 "A")
3. 버튼 클릭 → attribute 갱신 + localStorage 저장
4. custom.scss의 `[data-design-theme="X"]` 블록이 CSS 변수를 덮어써서 즉시 반영

# 사전지식

- **Quartz 컴포넌트 구조**: `.tsx` JSX + `.css` 변수 + `.inline.ts` 브라우저 스크립트 3점 세트. 컴포넌트 옆에 `scripts/`·`styles/` 폴더로 분리된다. 레퍼런스는 `quartz/components/Darkmode.tsx` 구조.
- **`document.addEventListener("nav", ...)` 패턴**: Quartz는 SPA 네비게이션을 지원해서 페이지 이동 시 DOM이 재구성된다. 이벤트 리스너는 `nav` 이벤트에서 다시 등록해야 한다. `window.addCleanup()`으로 해제도 잊지 말 것.
- **CSS 커스텀 프로퍼티 (CSS 변수) 상속**: `:root`에 정의한 `--light`는 모든 자손이 상속한다. `:root[data-design-theme="A"]`에서 재정의하면 같은 변수를 참조하는 수백 군데 스타일이 한 번에 바뀐다. Sass 변수(`$foo`)로는 불가능 — 컴파일 타임 값이라서.
- **Sass `@use` 규칙 순서**: `@use` 선언은 파일 최상단에 있어야 한다. CSS `@import url(...)`은 *컴파일 결과 CSS*에서 최상단이어야 한다. 이 두 제약이 충돌해서 외부 폰트 URL을 `@import`로 로드할 수 없다. 해결: Quartz config의 `typography` 슬롯으로 Google Fonts를 로드한다.
- **Google Fonts 3개 슬롯 제약**: `quartz.config.ts`의 `typography.{header, body, code}` 3개만 Google Fonts로 자동 로드된다. 3개 테마가 더 많은 폰트를 원해도 이 슬롯 안에서 해결해야 한다. 이번엔 IBM Plex Sans KR / Noto Serif KR / JetBrains Mono 3개를 로드해서 테마별로 CSS 변수로 조합.

# 전체 코드

## quartz/components/ThemeSwitcher.tsx

```tsx
// @ts-ignore
import themeSwitcherScript from "./scripts/themeswitcher.inline"
import styles from "./styles/themeswitcher.scss"
import { QuartzComponent, QuartzComponentConstructor, QuartzComponentProps } from "./types"
import { classNames } from "../util/lang"

const options = [
  { id: "A", label: "A", title: "Technical Docs — 깔끔/전문" },
  { id: "B", label: "B", title: "Academic — 학술/여백" },
  { id: "C", label: "C", title: "Developer — 다크/모노" },
]

const ThemeSwitcher: QuartzComponent = ({ displayClass }: QuartzComponentProps) => {
  return (
    <div
      class={classNames(displayClass, "theme-switcher")}
      role="group"
      aria-label="디자인 테마 선택"
    >
      {options.map((opt) => (
        <button
          class="theme-option"
          data-design={opt.id}
          title={opt.title}
          aria-label={opt.title}
        >
          {opt.label}
        </button>
      ))}
    </div>
  )
}

ThemeSwitcher.beforeDOMLoaded = themeSwitcherScript
ThemeSwitcher.css = styles

export default (() => ThemeSwitcher) satisfies QuartzComponentConstructor
```

## quartz/components/scripts/themeswitcher.inline.ts

```ts
const STORAGE_KEY = "design-theme"
const DEFAULT_DESIGN = "A"
const VALID = ["A", "B", "C"]

const initial = (() => {
  const saved = localStorage.getItem(STORAGE_KEY)
  return saved && VALID.includes(saved) ? saved : DEFAULT_DESIGN
})()
document.documentElement.setAttribute("data-design-theme", initial)

document.addEventListener("nav", () => {
  const syncActive = (design: string) => {
    document.querySelectorAll(".theme-option").forEach((el) => {
      const match = el.getAttribute("data-design") === design
      el.classList.toggle("active", match)
      el.setAttribute("aria-pressed", match ? "true" : "false")
    })
  }

  const current = localStorage.getItem(STORAGE_KEY) ?? DEFAULT_DESIGN
  document.documentElement.setAttribute("data-design-theme", current)
  syncActive(current)

  const switchDesign = (e: Event) => {
    const target = e.currentTarget as HTMLElement
    const design = target.getAttribute("data-design")
    if (!design || !VALID.includes(design)) return
    document.documentElement.setAttribute("data-design-theme", design)
    localStorage.setItem(STORAGE_KEY, design)
    syncActive(design)
  }

  for (const btn of document.querySelectorAll(".theme-option")) {
    btn.addEventListener("click", switchDesign)
    window.addCleanup(() => btn.removeEventListener("click", switchDesign))
  }
})
```

# 라인별 해설

## ThemeSwitcher.tsx

- `options` 배열을 분리한 이유: JSX 안에 data를 섞으면 나중에 테마 추가할 때 수정 포인트가 여러 곳에 흩어진다. 배열 1곳만 수정하면 렌더링이 자동 갱신되게 한다.
- `class={classNames(displayClass, "theme-switcher")}`: Quartz의 `displayClass`는 레이아웃 환경(모바일 only·데스크톱 only 등)을 표시하는 클래스. `classNames`로 병합해야 Flex 컴포넌트 안에서 반응형 표시가 정상 동작.
- `ThemeSwitcher.beforeDOMLoaded = themeSwitcherScript`: Quartz는 컴포넌트가 가진 `beforeDOMLoaded`·`afterDOMLoaded` 속성을 보고 브라우저로 내려보낼 inline 스크립트를 주입한다. 번들러 설정 추가 없이 컴포넌트 단위로 스크립트가 함께 배포된다.
- `ThemeSwitcher.css = styles`: 컴포넌트 전용 CSS도 같은 방식. scss 파일을 ESM import하면 Quartz 빌드가 css 문자열로 변환해준다.

## themeswitcher.inline.ts

- 파일 최상단의 `document.documentElement.setAttribute("data-design-theme", initial)`: **nav 이벤트 밖**에서 실행되는 이유는 **FOUC 방지**. 페이지 로드 최초 순간에 즉시 attribute를 붙여야 CSS가 첫 렌더부터 올바른 테마로 그려진다. nav 이벤트는 DOM 렌더 이후에 발동해서 늦다.
- `document.addEventListener("nav", ...)`: Quartz SPA 네비게이션 시 DOM이 교체되면 버튼 엘리먼트도 새로 생긴다. 매번 이 이벤트 안에서 리스너를 재등록해야 한다.
- `syncActive()`: active 클래스와 `aria-pressed` 둘 다 갱신. aria-pressed는 접근성(스크린 리더) 대응.
- `window.addCleanup(() => ...)`: Quartz가 제공하는 유틸. 다음 nav 이벤트 시 자동 호출되어 리스너를 제거한다. 메모리 누수 방지.
- `VALID` 화이트리스트 검사: localStorage는 사용자가 임의로 변경할 수 있다. `"D"` 같은 값이 들어와도 기본값 fallback.

## themeswitcher.scss

- `border-radius: 999px`로 pill 형태. 버튼 3개를 감싸는 group도 pill이고 각 버튼도 원형이어서 시각적으로 일체감.
- `var(--codeFont, monospace)`: 버튼 라벨 "A/B/C"를 code 폰트로 표시. fallback이 있어서 code 폰트가 로드되지 않아도 안전.
- `&.active { background: var(--secondary); color: var(--light); }`: 활성 버튼은 accent 컬러로. `--secondary`는 테마마다 다르게 정의되어 있어서, 스위처 버튼 자체도 현재 테마의 accent 컬러를 반영한다.

## custom.scss 테마 블록

- `:root[data-design-theme="A"] { ... }`: CSS 변수를 여기서 재정의하면 Quartz의 `base.scss`가 쓰는 모든 `var(--light)`·`var(--secondary)` 참조가 이 값을 가져간다. base.scss 수정 없이 테마 전환 가능.
- `:root[data-design-theme="A"][saved-theme="dark"] { ... }`: 디자인 A의 다크 모드. attribute 2개가 동시에 매칭되어야 적용. 다크모드 토글(Darkmode.tsx)은 이미 `saved-theme`을 갱신하고 있어서 그대로 활용.
- `--shadow-soft: none`: A는 플랫 디자인이라 그림자 제거. B에선 `0 8px 26px rgba(60, 40, 20, 0.07)`로 부활.
- Windows 환경 주의: `@import url(...)`로 외부 폰트를 가져오려 했다가 실패했다. Sass `@use`와 CSS `@import` 순서 제약이 충돌해서다. Quartz config의 `typography` 슬롯을 사용하는 것이 해법.

## quartz.config.ts 폰트 교체

- `typography` 안의 3개 슬롯(header/body/code)은 Google Fonts에서 빌드 시 로드된다. 테마 A/B/C가 **이 3개 폰트를 조합해서** 사용하도록 설계. 4번째 폰트는 불가능.
- 한글 폰트를 골고루: IBM Plex Sans KR(sans), Noto Serif KR(serif), JetBrains Mono(mono). 일반적으로 Quartz는 한 가지 한글 폰트만 쓰는데 테마별 차이를 내려면 이렇게 다변화해야 한다.

# 교훈

- **CSS 변수 오버라이드가 테마 전환의 교과서적 해법이다.** 테마마다 컴포넌트 분기·클래스 분기 하면 유지보수가 망가진다. 변수 덮어쓰기 한 판이면 끝.
- **FOUC 방지는 inline 스크립트를 `<head>`에 넣고 nav 이벤트 이전에 실행되게 해야 한다.** Quartz의 `beforeDOMLoaded` 슬롯이 이 역할을 한다. 버튼 리스너는 nav 이벤트 안에서 등록하지만, 초기 attribute 세팅은 그 밖에서 해야 한다.
- **Sass `@use`와 CSS `@import` 순서 제약은 실전에서 자주 부딪힌다.** 외부 폰트는 `@import url`이 아니라 프레임워크가 제공하는 폰트 로더 슬롯으로 넣는 게 마찰이 적다.
- **Quartz의 3개 폰트 슬롯 제약**은 테마별 폰트 다양화의 상한선이다. 디자인 3안 정도가 한계치. 넘어가려면 `Head.tsx`에 직접 `<link>` 태그를 주입하거나 커스텀 plugin을 만들어야 한다. 코어 미수정 원칙을 고려하면 3개에서 멈추는 게 합리.
- **디자인은 실제 콘텐츠 위에 올려서 비교해야 정확하다.** 3안을 평행 배치해서 실제 본문·코드블록·제목과 함께 보면 Figma만 볼 때보다 훨씬 빨리 결론이 난다.
- **Quartz 코어(`quartz/`) 수정은 Docker dev 모드를 켜야 반영된다.** 이번 작업 중간에 Docker 볼륨 마운트 구조 때문에 한참 헤맸다. 상세: [[1. Docker 개발 모드]]
